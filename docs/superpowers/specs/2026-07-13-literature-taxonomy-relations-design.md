# 文献分类与演化关系重建设计

## 目标

以 86 篇正式发表文献为粒度，重新核定年份位置、主泳道、文献类型和演化关系，使时间线表达“研究贡献如何演进”，而不是把共同作者、相似应用或关键词重合误画为继承关系。

## 证据顺序

1. `../03_文献梳理/08_文献分类总表.md` 的逐篇分类与摘要。
2. 同目录 `01` 至 `06` 分册中明确列出的算法族、模型族与阶段演进。
3. `papers.csv` 中的正式年份、题名、期刊、作者与 DOI。
4. 对仍无法确认的关系采用宁缺毋滥原则，不显示连线。

## 位置规则

- 横坐标由正式发表年份唯一决定。
- 泳道由论文主要贡献决定，实验应用不覆盖算法或理论主贡献。
- `fixed-point`：不动点、分裂可行性、分裂公共不动点和 CQ/投影方法。
- `imaging`：主要贡献是图像模型、成像任务或稀疏重建模型。
- `primal-dual`：原始对偶、PDFP、ADMM/AMA 与以对偶结构为主的算法。
- `monotone-vi`：单调包含、三/四算子分裂、FBHF、变分不等式与 Banach 推广。
- `statistics`：函数型回归、变量选择和空间自回归统计模型。
- `intelligent-systems`：机器人视觉、传感网络、组合计算、数值 PDE 和其他交叉计算工作。

## 文献类型规则

- `theory`：主要结果是算子性质、存在性、强弱收敛或误差理论。
- `algorithm`：提出新的基础算法框架。
- `extension`：对已有框架加入惯性、松弛、预条件、变度量、随机化或参数改造。
- `application`：主要创新是具体图像、统计或交叉应用模型。

## 关系规则

- `parent_id` 表示父文献到当前文献的研究演化边。
- 父文献年份不得晚于子文献年份；同年仅保留本地材料明确说明的正式延续关系。
- 只保留同一算法族、同一模型族或明确方法继承关系。
- 共同作者、研究方向相近、都包含图像实验，不足以构成演化关系。
- 关系图保持稀疏，孤立节点允许存在。

## 验收

- 86 个 ID 唯一，年份、方向和类型合法。
- 所有父 ID 存在，无自环、无重复边、无未来父文献。
- 明确错误关系 A3→A11、A52→A54、A61→A67 不再存在。
- 明确主线可追踪：FBHF、三算子、原始对偶、SFP、图像模型、统计模型。
- Chrome 中节点位于正确年份和泳道，点击详情显示的前序/后续与连线一致。
