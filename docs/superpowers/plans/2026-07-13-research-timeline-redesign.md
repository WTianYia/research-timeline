# 研究时间线实现计划

1. 抽离纯数据模块，先用 Node 内置测试覆盖 CSV 解析、规范化、组合筛选、关系统计和阶段摘要。
2. 重构 `index.html` 为顶部工具栏、左侧导航、中央 SVG 画布、右侧详情抽屉和底部密度导航。
3. 重写 `css/style.css`，匹配 Stitch 视觉基线，并补齐桌面、平板和移动端布局。
4. 重写 `js/timeline.js`，接入真实数据、视窗缩放平移、节点选择、详情、关系高亮、筛选和导出。
5. 运行自动化测试与静态检查，在本地 HTTP 服务中验证全部核心交互。
6. 进行源截图与实现截图并排视觉 QA，修复可见差异并记录到 `design-qa.md`。
7. 提交到 `main`、推送 GitHub，验证 Cloudflare Pages 默认域名和正式域名。
